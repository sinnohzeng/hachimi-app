# 文案规范指南

> 基于 Ant Design 设计规范整合的文案书写规范。

---

## 一、Voice 和 Tone

### 1.1 Voice 声音

作为现代、轻量的设计系统，我们希望用高效的表达为用户留下愉悦印象。三个原则：

1. **直接清晰**：简明扼要地传达信息，帮助用户高效完成任务
2. **真实友好**：像人一样说话，富有同理心，与用户站在一起
3. **乐于助人**：预测用户的真实需求，在恰当的时候提供帮助与信息

### 1.2 Tone 语调

根据不同情景调整语调：

| 语调类型 | 适用场景 | 示例 |
|----------|----------|------|
| **中性的** | 用户快速完成日常工作 | 将图像添加到空白部分以开始构建您的网站 |
| **庆祝的** | 完成复杂任务 | 恭喜！你已成为 AnyWeb 的一员 |
| **同情的** | 用户看到错误消息 | URL 不可用。可联系站点管理员帮助解决 |
| **有帮助的** | 引导用户 | 准备好创建你的第一个网站了吗？跟随 3 步小妙招开始吧 |
| **权威的** | 违反政策 | 因违反政策，你的网站「每日咖啡新闻」已被删除 |

## 二、书写原则

### 2.1 简洁清晰

只需要提供给用户必要的信息，省略多余的废话。

| ✅ 推荐用法 | ❌ 不推荐用法 |
| --- | --- |
| 不知道如何使用吗？联系我们 | 如何使用 SOP 配置有问题？不用担心！您可以点击下面的图标与我们联系... |

### 2.2 像人一样说话

用最容易理解的最朴素的文字展示信息。

| ✅ 推荐用法 | ❌ 不推荐用法 |
| --- | --- |
| 网页出错了，请稍后重试 | 404 ERROR |
| 视频加载中... | Feed 流 Skeleton 加载中... |

### 2.3 开门见山，直击重点

优先展示最重要的信息。

| ✅ 推荐用法 | ❌ 不推荐用法 |
| --- | --- |
| 3 步创建工单 | 创建你的工单需要经过 3 个步骤 |
| 文件已上传 | 你已经成功上传了文件 |

### 2.4 保持一致

使用一致的术语：

| 中文 | English | ❌ 不推荐用法 |
| --- | --- | --- |
| 你 | You | 您 |
| 账号 | Account | 帐号 |
| 登录 | Log in | 登陆、Login（仅用作名词） |
| 新建 | Create | 创建 |
| 添加 | Add | 新增 |
| 删除 | Delete | 清除 |
| 移除 | Remove | - |
| 启用 | Enable | 开启、开始使用 |
| 停用 | Disable | 暂停、停止使用 |
| 其他 | Other | 其它 |
| 注册 | Create account | Register、Sign up |
| 链接 | URL | Link |

### 2.5 与用户平起平坐

- 尽量避免使用请求和道歉的语气，只有非常严重的情况才使用"对不起"
- 避免用教育、指责和埋怨的语气

| ✅ 推荐用法 | ❌ 不推荐用法 |
| --- | --- |
| 点击"编辑"修改内容 | 对不起，您无法编辑表单，因为您处于查看模式... |

### 2.6 避免夸大其词

尽量避免使用绝对词，例如"最好"、"绝对"、"总是"等等。

### 2.7 具有包容性

- 不要使用描述年龄的词，如"年轻"、"老"等
- 不使用具有明显性别倾向的词
- 不要使用黑名单或白名单，改用"禁用名单"
- 考虑色盲群体，不使用颜色描述按钮
- 不使用感官动词，专注于任务的最终目标

## 三、语法规范

### 3.1 大小写

**Sentence case**（大多数情况）：

| ✅ 推荐用法 | ❌ 不推荐用法 |
| --- | --- |
| Create project | Create a Project |
| 2 results found | 2 Results found |
| Country/Region | Country/region |

**Title Case**（专有名词）：

- 品牌名、平台名、功能名、商标等须单词首字母大写
- 人名前后跟随工作职位时须单词首字母大写

### 3.2 缩略式

推荐使用缩略式营造轻松语调：

| ✅ 推荐用法 | ❌ 不推荐用法 |
| --- | --- |
| Can't, don't, it's | cannot, do not, it is |

但在敏感或严肃信息上避免使用缩略式。

### 3.3 日期与时间

- 使用 24 小时制表示时刻，如 07:00，21:00
- 日期格式：`Jan 1, 2018` 或 `2018-01-01`
- 使用短线 `–` 连接年月日，不使用 `/`
- 时间范围使用 `~`，如：`2020-02-06 ~ 2020-02-08`

**时间戳格式**：

| 时间范围 | 显示格式 |
| --- | --- |
| 少于 1 分钟 | Just now |
| 1–60 分钟 | 13 minutes ago |
| 今天 | 10:30 |
| 昨天 | Yesterday at 10:30 |
| 最近 7 天 | Friday at 10:30 |
| 7天–1年 | Aug 14 at 10:30 |
| 超过 1 年 | Aug 14, 2016 at 10:30 |

### 3.4 数字

- 四位数及更多时，使用逗号隔开：`$2,353.56`，`5,345 MB`
- 数字和单位之间需要有空格：`5,345 MB`，`7 lb`
- 数字范围用横杠分隔，前后不加空格：`2:00 pm-3:00 pm`

### 3.5 列表

- 列表内容小于等于六条
- 以介绍性语句开始，并以冒号":"结尾
- 使用平行语句
- 首单词首字母大写
- 结尾不使用任何标点符号

## 四、标点符号

### 4.1 和号

除专有名词外，不要使用"&"，用"and"代替。

### 4.2 冒号

- 冒号后的首个单词首字母不需要大写（除非是专业术语）
- 使用冒号引入列表

### 4.3 逗号

- 使用牛津逗号（oxford comma）
- 不要使用逗号分隔项目符号或编号的列表项

### 4.4 破折号和连字符

- 破折号展示数字范围，数字前后不带空格：`23–52`
- 连字符连接组合形容词：`14-day trial`

### 4.5 省略号

- 动作正在进行中时使用：`uploading...`
- 组件有限宽度下无法展示所有文案时做截断

### 4.6 句号

- 以下场景不用句号：按钮、一句话的 Toast、占位符、标题、通知、导航栏菜单、tooltip、radio、checkbox

### 4.7 引号

使用引号引用定义的名词或用户输入的信息：

| ✅ 推荐用法 | ❌ 不推荐用法 |
| --- | --- |
| Delete "john@example.com"? | Delete john@example? |

### 4.8 分号

**不要使用分号**，尝试将一句话拆分成多个较短的句子。

---

## 五、代词

- 大多数情况下，使用第二人称（"you"）与用户交流
- 避免使用第一人称（"I"或"my"）
- 避免不必要的代词

| ✅ 推荐用法 | ❌ 不推荐用法 |
| --- | --- |
| Edit profile | Edit my profile |

---

## 六、UI 元素

当文案指代产品中其他的 UI 元素时，需要**加粗**该文案。

> 例：Remove from **Saved**

不建议使用引号（""）表达 UI 元素，除非没有办法使用加粗效果。

---

## 七、表情

避免使用 emoji，选择用 icon 代替；如必须使用，可放在句段结尾。
