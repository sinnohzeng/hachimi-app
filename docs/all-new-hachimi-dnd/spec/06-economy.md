## 1. 概述

Hachimi v2.0 采用**双货币体系**：

- **金币（Gold）**：即时正反馈货币。通过专注时间获取，消费于配件、装饰、互动道具和临时加成。核心设计理念：**每次专注后都有东西可以买**。
- **星尘（Stardust）**：冒险层货币。通过骰子检定获取，消费于冒险专属配件和主哈基米物品。核心设计理念：**冒险的长期目标**。

两种货币**相互独立**，不可互相兑换。

### 设计哲学

金币不追求"稀缺性"，而追求"消费频率"。通胀本身不是问题 — **没有足够多的消费品来花掉金币**才是问题。因此金币经济的核心是：丰富的消费场景 + 分层定价 + 永远有下一个想买的东西。

## 2. 金币经济

### 2.1 收入端

| 来源 | 触发条件 | 金币数量 | 频率 |
|------|----------|---------|------|
| 专注时间 | 每分钟专注 | 2 🪙 | 每次专注 |
| 每日签到 | 首次打开 App | 30 🪙 | 每日 1 次 |
| Full House | 当日所有习惯完成 | 20 🪙 | 每日最多 1 次 |
| 成就解锁 | 解锁成就 | 50-200 🪙 | 不定期 |
| 骰子溢出 | 骰子超过 20 枚上限 | 5 🪙/枚 | 不定期 |

### 2.2 日收入估算

| 用户类型 | 日专注 | 日金币 | 说明 |
|---------|--------|--------|------|
| 轻度（1×25min） | 25 min | ~80 🪙 | 50(专注) + 30(签到) |
| 中度（3×25min） | 75 min | ~200 🪙 | 150(专注) + 30(签到) + 20(Full House) |
| 重度（5×25min） | 125 min | ~280 🪙 | 250(专注) + 30(签到) + 20(Full House) |

**设计目标**：中度用户日均 ~200 金币。

### 2.3 支出端 — 三档定价模型

#### 第一档：即时消耗（15-50 金币）

每次专注后立刻能买。**核心正反馈体验。**

| 物品 | 价格 | 效果 | 消费场景 |
|------|------|------|---------|
| 猫罐头 🥫 | 15 🪙 | 猫咪播放"吃饭"动画 + 心情 +1 | 酒馆互动 |
| 小鱼干 🐟 | 20 🪙 | 猫咪播放"开心"动画 + 心情 +1 | 酒馆互动 |
| 毛线团 🧶 | 25 🪙 | 猫咪播放"玩耍"动画（Rive 状态机） | 酒馆互动 |
| 检定 +2 加成 ⚔️ | 30 🪙 | 下一次骰子检定修正值 +2（一次性） | 冒险系统 |
| 额外骰子 🎲 | 50 🪙 | 获得 1 枚额外待投骰子（不受上限 20 限制） | 冒险系统 |

> **互动道具使用场景**：在 Tab 2（酒馆）中，点击猫咪 → 弹出互动菜单 → 选择食物/玩具 → 猫咪播放对应 Rive 动画。

#### 第二档：日常配件（100-300 金币）

1-2 天储蓄即可购买。**收集驱动。**

| 物品 | 价格 | 说明 |
|------|------|------|
| 普通帽子 🎩 | 100 🪙 | 各种风格的小帽子 |
| 项圈 📿 | 150 🪙 | 铃铛/领结/蝴蝶结 |
| 围巾 🧣 | 200 🪙 | 季节性围巾 |
| 小装饰品 ✨ | 250 🪙 | 花朵/羽毛/蝴蝶 |
| 背包 🎒 | 300 🪙 | 迷你冒险背包 |

> 有了 Rive 后，每只猫可以同时装备多件配件（头部+颈部+背部），不再限制为单件。

#### 第三档：稀有收藏（500-1000 金币）

3-5 天储蓄。**长期目标。**

| 物品 | 价格 | 说明 |
|------|------|------|
| 酒馆灯笼 🏮 | 500 🪙 | 酒馆场景装饰（Flame 场景中可见） |
| 酒馆地毯 🪴 | 500 🪙 | 酒馆场景装饰 |
| 外观微调 🎨 | 500 🪙 | 重新随机猫咪外观参数 |
| 稀有配件（皇冠） 👑 | 600 🪙 | 限量配件 |
| 酒馆壁画 🖼️ | 800 🪙 | 酒馆场景装饰 |
| 传奇家具（壁炉升级）🔥 | 1000 🪙 | 酒馆核心装饰，视觉效果显著 |

### 2.4 节奏验证

## 3. 星尘经济

星尘定价已经过验证，Phase 2 实施时直接采用。

### 3.1 星尘获取

| 来源 | 触发条件 | 星尘数量 |
|------|----------|---------|
| 骰子大成功（nat 20 或超 DC 10+）| 检定结算 | 15 ⭐ |
| 骰子成功（≥ DC）| 检定结算 | 8 ⭐ |
| 骰子失败（< DC）| 检定结算 | 3 ⭐ |
| 骰子大失败（nat 1）| 检定结算 | 1 ⭐ |
| 场景完成（★★★）| 冒险结算 | 额外 50% 星尘 |

### 3.2 日收入估算

- 每天 3 次专注 → 3 枚骰子 → 期望 ~16.5 ⭐/天

### 3.3 星尘定价表

| 物品 | 价格 | 储蓄天数 |
|------|------|---------|
| 普通冒险配件 | 50 ⭐ | ~3 天 |
| 精品冒险配件 | 120 ⭐ | ~7 天 |
| 主哈基米专属配件 | 200 ⭐ | ~12 天 |
| 骰子外观 | 80 ⭐ | ~5 天 |
| 传奇头冠 | 350 ⭐ | ~21 天 |

### 3.4 持久化

- `materialized_state` 表的 `stardust` 字段
- Firestore：`users/{uid}/materializedState.stardust`

## 4. 双货币体系对比

| 维度 | 金币（Gold）| 星尘（Stardust）|
|------|-----------|----------------|
| 设计理念 | 即时正反馈 | 长期目标 |
| 主要来源 | 专注时间（2/min）+ 签到 | 骰子检定 |
| 消费节奏 | 每次专注后就能花 | 每周 1-2 件 |
| 消费对象 | 互动道具 + 配件 + 酒馆装饰 + buff | 冒险配件 + 骰子外观 |
| 持久化 | `materialized_state.gold` | `materialized_state.stardust` |
| 可兑换 | 否 | 否 |
| Remote Config | 是（收入率可调） | 否（固定） |

## 5. Remote Config 参数

所有金币收入参数通过 Firebase Remote Config 控制，支持无需更新 App 的动态调整：

```
gold_income_rate_per_minute = 2
gold_checkin_reward = 30
gold_full_house_reward = 20
gold_achievement_reward_small = 50
gold_achievement_reward_large = 200
gold_dice_overflow_rate = 5
```

物品价格暂时硬编码（Phase 3 实施时评估是否需要 Remote Config）。

## 6. 用户等级 XP

> 此节与 `spec/05-party-and-class.md` §9 保持一致，此处为经济视角补充。

### 6.1 XP 计算

```dart
int get adventureLevel => allCats.fold(0, (sum, cat) => sum + cat.totalMinutes);
```

- 1 专注分钟 = 1 XP（冒险等级 XP，与现有 `xp_service.dart` 独立）
- 完整 20 级 XP 表见 [spec/05-party-and-class.md](05-party-and-class.md)

### 6.2 与现有 XP 系统的区别

| 维度 | 冒险等级 XP | 现有 `xp_service.dart` |
|------|------------|----------------------|
| 计算 | 原始分钟数 | 分钟 × 倍率 + fullHouseBonus |
| 用途 | DnD 等级 + 区域解锁 | 成就系统 |
| 交叉影响 | 无 | 无 |

## 7. 经济平衡验证

### 7.1 模拟测试

Phase 3 实施后执行：

| 场景 | 验证点 |
|------|--------|
| 轻度用户 30 天 | 金币累计 ~2,400。可买 ~8 件日常配件 + 4 件稀有。合理。|
| 中度用户 30 天 | 金币累计 ~6,000。可买 ~20 件日常 + 5 件稀有 + 大量互动。合理。|
| 重度用户 30 天 | 金币累计 ~8,400。物品种类可能不够花 → 需要 Phase 3 补充更多消费品。|
| 极端用户（10×/天）| 金币累计 ~15,000。需要第四档物品（2000+ 金币）或金币 → 星尘转换。|

### 7.2 Remote Config 调整流程

```
发现经济失衡
  → 更新 Firebase Remote Config
  → 5 分钟内生效（下次 App 前台时拉取）
  → 监控 7 天
  → 确认后更新 spec/06 定稿值
```

## 8. 验收标准

### 金币

- [ ] 专注完成后金币按 2/min 正确计入
- [ ] 每日签到金币 30 正确发放（仅首次）
- [ ] Full House 奖励 20 正确判定（当日全部习惯完成）
- [ ] 金币持久化在 `materialized_state.gold`，重启不丢失
- [ ] 金币收入率通过 Remote Config 控制，修改后下次前台生效
- [ ] 互动道具在酒馆中正确触发猫咪 Rive 动画
- [ ] 检定 +2 buff 正确叠加到下次骰子检定
- [ ] 额外骰子正确添加到 pending_dice（不受 20 上限限制）

### 星尘

- [ ] 检定结算后星尘按奖励表正确增加
- [ ] 星尘持久化在 `materialized_state.stardust`
- [ ] ★★★ 场景完成额外 50% 星尘正确计算

### 通用

- [ ] 金币和星尘在 UI 上分别显示，不混淆
- [ ] 两种货币增减操作相互独立
- [ ] 购买物品时余额不足有明确提示（不是静默失败）
- [ ] 冒险等级与 `xp_service.dart` 结果独立
