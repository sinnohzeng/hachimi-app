## 1. 概述

Hachimi v2.0 采用**双货币体系**：

- **金币（Gold）**：通过专注时间、签到、成就获取，消费于基础配件和酒馆装饰。属于"工具层"货币，反映日常使用行为。
- **星尘（Stardust）**：通过骰子检定、场景冒险完成获取，消费于冒险专属配件和主哈基米专属物品。属于"冒险层"货币，与 DnD 机制深度绑定。

两种货币**相互独立**，获取一种不影响另一种，不可相互兑换。

## 2. 星尘（Stardust）经济

星尘定价已经过验证，Phase 2 实施时直接采用。

### 2.1 星尘获取来源

| 来源 | 触发条件 | 星尘数量 |
|------|----------|---------|
| 骰子大成功（nat 20 或超 DC 10+）| 检定结算 | 15 ⭐ |
| 骰子成功（≥ DC）| 检定结算 | 8 ⭐ |
| 骰子失败（< DC）| 检定结算 | 3 ⭐ |
| 骰子大失败（nat 1）| 检定结算 | 1 ⭐ |
| 场景完成（★★★）| 冒险结算 | 额外 50% 星尘（基于本次检定累计） |

### 2.2 日收入估算

- 假设每天 3 次专注 → 3 枚骰子
- 平均投掷结果：成功率约 55%（考虑属性和 DC 分布）
- 期望值：3 × (0.1×15 + 0.45×8 + 0.4×3 + 0.05×1) ≈ 3 × 5.5 = **~16.5 ⭐/天**

### 2.3 星尘定价表

| 物品 | 价格 | 储蓄天数（@16.5⭐/天）|
|------|------|---------------------|
| 普通冒险配件 | 50 ⭐ | ~3 天 |
| 精品冒险配件 | 120 ⭐ | ~7 天 |
| 主哈基米专属配件 | 200 ⭐ | ~12 天 |
| 骰子外观 | 80 ⭐ | ~5 天 |
| 传奇头冠 | 350 ⭐ | ~21 天 |

设计目标：日常物品 3–7 天可购，稀有物品 12–21 天，创造有意义的目标感而不令人沮丧。

### 2.4 星尘持久化

- 存储于 `materialized_state` 表的 `stardust` 字段（整数）
- Firestore 路径：`users/{uid}/materializedState.stardust`
- 每次检定结算后原子更新，遵循现有 Ledger 模式

## 3. ⚠️ [待定稿] 金币经济

> **此节需要在 Phase 3 实施前完成平衡决策，方可定稿。**

### 3.1 问题描述

当前代码（`gold_service.dart`）的金币收入率为 **10 金币/分钟**。经分析，此速率导致严重通货膨胀：

- 25 分钟专注 → 250 金币
- 多数物品定价 50–500 金币
- 用户几天内即可购买全部基础物品，金币失去激励意义

### 3.2 参考模型对比

| 模型 | 收入率 | 日收入（3×25min）| 500 金币物品需储蓄 | 来源 |
|------|--------|-----------------|-------------------|------|
| 当前实现 | 10/min | ~750 金币 | <1 天 | 现有代码 |
| 研究方案 | 1/min | ~80–130 金币 | 5–8 天 | 综合分析 §10 |

### 3.3 决策原则

最终收入率应满足：**高价值物品（500 金币）需要 5–10 天的日常专注**。

待定参数清单（Phase 3 实施时确认）：

- [ ] 每分钟金币收入率：**[待定]**（候选：1/min 或 2/min）
- [ ] 每次签到金币奖励：**[待定]**（候选：20 或 50）
- [ ] 成就解锁金币奖励：**[待定]**（候选：100 或 200）
- [ ] 骰子堆积上限溢出转化率：5 金币/枚（已确认，见 spec/03）
- [ ] 外观微调费用：500 金币（已确认，见 spec/01）

### 3.4 [待定] 金币定价占位表

> 以下价格为占位值，Phase 3 平衡测试后更新。

| 物品类别 | 价格范围（占位）|
|----------|---------------|
| 基础配件 | **[待定]** 50–350 金币 |
| 酒馆装饰 | **[待定]** 100–500 金币 |
| 外观微调（一次性重随）| 500 金币（已确认）|

### 3.5 Remote Config 支持

金币收入率和物品价格必须通过 **Firebase Remote Config** 配置，支持无需 App 更新的动态调整。Remote Config 键名：

```
gold_income_rate_per_minute
gold_checkin_reward
gold_achievement_reward_small
gold_achievement_reward_large
```

## 5. 用户等级 XP

> 此节与 `spec/05-party-and-class.md` §9 保持一致，此处为经济视角的补充说明。

### 5.1 XP 计算

```dart
int get userXP => allCats.fold(0, (sum, cat) => sum + cat.totalMinutes);
```

- **1 专注分钟 = 1 XP**（应用职业加成后的分钟数）
- XP 来源：所有猫（PrimaryCat + 全部 CompanionCat）的 `totalMinutes` 之和

### 5.2 与现有 XP 系统的区别

| 维度 | DnD 等级 XP | 现有 `xp_service.dart` XP |
|------|------------|--------------------------|
| 计算基础 | 原始专注分钟数 | 分钟数 × 倍率（Remote Config）+ fullHouseBonus |
| 用途 | DnD 等级 + 职业解锁 | 成就系统、猫咪等级显示 |
| 跨系统影响 | 无 | 无 |

两套 XP 体系**完全独立**，不互相干扰。

### 5.3 等级阈值（完整表）

| 等级段 | 累计 XP（分钟）| 熟练加值 | 解锁 |
|--------|--------------|---------|------|
| 1–4 | 0–2,000 | +2 | 基础场景（猫咪小镇） |
| 5–8 | 2,000–8,000 | +3 | 迷雾森林 |
| 9–12 | 8,000–20,000 | +4 | 远古遗迹 |
| 13–16 | 20,000–50,000 | +5 | 传奇场景 |
| 17–20 | 50,000–100,000 | +6 | 全部内容 |

等级 3 触发职业选择，等级 10 触发多职兼修（详见 `spec/05-party-and-class.md`）。

## 6. 经济平衡验证计划

Phase 3 实施金币经济后，需执行以下验证：

### 6.1 模拟测试

- 模拟 30 天、日均 3 次专注（各 25 分钟）场景：
  - 金币累计 vs. 物品价格梯度是否匹配
  - 星尘累计是否符合 §2.2 估算
- 模拟极端用户（1 次/天 vs. 10 次/天）的经济体验差距

### 6.2 Remote Config 调整流程

```
发现经济失衡
  → 更新 Firebase Remote Config 参数
  → 无需 App 更新，5 分钟内生效（下次 App 前台时拉取）
  → 监控 7 天数据
  → 确认后在 spec/06-economy.md 中更新为定稿值
```

### 6.3 不需要 A/B 测试

当前无真实用户，Phase 3 测试在开发者设备上验证即可，不需要 Firebase A/B Testing。
