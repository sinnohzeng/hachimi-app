## 1. 产品定位与核心价值

### 一句话定位

**Hachimi 是一款把「养猫」和「专注计时」深度融合的习惯养成 App：用户通过完成专注习惯，养活一只独一无二的像素虚拟猫。**

### 核心问题

传统习惯追踪 App 的激励机制太抽象。「今天专注了 45 分钟」「连续打卡 7 天」这些数字对大多数用户来说缺乏情感共鸣，难以形成长期黏性。

### 解决方案

Hachimi 把每一分钟的专注，变成一只具体的、有名字、有性格、有心情的猫咪的成长。用户「照顾」的不是一个抽象的习惯指标，而是一只在乎他们的猫。

### 产品差异点

| 维度 | Hachimi | 传统习惯 App | 宠物养成游戏 |
|------|---------|------------|------------|
| 习惯追踪 | ✅ | ✅ | ❌ |
| 情感连接 | ✅ 强（猫咪 × 习惯绑定） | ❌ 弱 | ✅ |
| AI 互动 | ✅ 日记 + 聊天 | ❌ | 部分 |
| 离线可用 | ✅ 完全离线 | 部分 | 部分 |
| 外观个性化 | ✅ 像素猫 14 参数随机生成 | ❌ | ✅ |

## 2. 核心产品循环

```
创建习惯
    ↓
领养一只像素猫（绑定该习惯）
    ↓
开始专注计时（倒计时 or 秒表）
    ↓
完成专注 → 获得金币 + XP + 猫咪成长时间
    ↓
猫咪成长（4 个阶段：幼猫 → 少年 → 成年 → 老年）
    ↓
AI 自动生成猫咪今日日记（猫咪视角，2-4 句）
    ↓
用金币购买配饰装扮猫咪
    ↓
解锁成就 + 获得称号
```

这个循环的核心是：**每次专注都有具体可见的结果**（猫咪成长了多少分钟），而不是冷冰冰的数字累加。

## 3. 目标用户画像

### 核心用户群

- **年龄**：18-35 岁
- **特征**：有明确学习目标（语言、编程、创作、健身等），对可视化反馈和轻度游戏化有好感，长期导向
- **动机**：希望建立稳定的专注习惯，但纯番茄钟或纯习惯打卡 App 坚持不下去

### 用户场景举例

- 备考学生：把「考研数学」习惯绑定一只猫，每天计时复习，看着猫咪从幼猫长到少年
- 自由职业者：每天靠 Hachimi 进入工作状态，猫咪的心情提醒他「你已经 3 天没专注了」
- 语言学习者：每个语言绑定一只猫，多只猫对应多个学习目标，猫咪室里直观看到每个目标的投入程度

### 非目标用户

- 需要严格番茄钟节律（25/5 模式）的用户
- 需要社交打卡、互相监督功能的用户

## 4. 功能全景

### 4.1 习惯管理

**创建习惯**

- 名称（自由输入）+ emoji 图标（30 个预设可选）
- 每日专注目标时长：15 / 25 / 40 / 60 分钟，或自定义
- 提醒时间：最多 5 个，支持快捷时间选择
- 总目标时长（可选）：设置总目标小时数，达成后自动转为永续模式

**习惯详情页**

- 查看该习惯的历史会话记录
- 专注热力图（日历格式）
- 累计时长、累计打卡天数
- 进度百分比（设置了总目标时才有）

**习惯状态**

- 活跃（Active）：正常使用
- 已完成（Target Completed）：总目标达成后转为永续，持续积累时间
- 猫咪随习惯状态绑定，习惯停用时猫咪进入休眠

---

### 4.2 专注计时器

**两种计时模式**

| 模式 | 说明 | 进度环 |
|------|------|--------|
| 倒计时（Countdown） | 设定目标时长，时间耗尽则完成 | 顺时针减少 |
| 正计时（Stopwatch） | 无限计时，到达每日目标后自动标记完成 | 顺时针增加，满圆为目标完成 |

**计时器界面**

- 大号时间显示居中
- 绑定猫咪的像素精灵作为主视觉（计时时猫咪「陪伴」用户）
- 习惯连续天数徽章
- 「放弃」按钮（需长按，防止误触）
- 「暂停」按钮

**后台行为**

| 场景 | 行为 |
|------|------|
| 切到后台 < 15 秒 | 继续计时，无感知 |
| 切到后台 15 秒到 5 分钟 | 自动暂停，回来时提示 |
| 切到后台 > 5 分钟 | 自动完成，记录实际专注时间 |

**后台保活**：Android 前景服务（Foreground Service），持久通知显示「猫咪 emoji + 习惯名称 + 当前时间」，即使手机锁屏也持续计时。

**放弃会话规则**

- 放弃时已计时 < 5 分钟：丢弃，无奖励
- 放弃时已计时 ≥ 5 分钟：保存为「放弃」会话，记录实际时间，无阶段升级加成

---

### 4.3 专注完成体验

完成一次专注会话后，进入专注完成页（FocusCompleteScreen）：

- **彩纸动画**：2 秒爆发，30 个粒子，5 种颜色
- **振动反馈**：完成（重振动：200ms + 100ms + 300ms）/ 放弃（轻振动：100ms）
- **奖励明细卡**：
  - 基础 XP（= 专注分钟数 × 1）
  - 连续天数加成（≥3 天：+5 XP）
  - 里程碑加成（7/14/30 天：+30 XP）
  - 全部完成加成（今日所有习惯都完成：+10 XP）
  - 总计 XP
  - 获得金币数
- **猫咪升级提示**（如果本次专注触发了阶段升级）
- **AI 日记**（异步触发生成，fire-and-forget，不阻塞完成页）

### 4.4 猫咪养成系统

**领养流程（3 步）**

1. **习惯配置**：填写习惯名称、目标、提醒
2. **选择猫咪**：随机生成 3 只候选猫咪（各有不同性格和外观预览），可刷新重新随机，点击选择
3. **命名猫咪**：输入名字（有随机默认名可直接采用），点击「领养」

**6 种性格（创建时随机分配，不可更改）**

| 性格 | Emoji | 说明 |
|------|-------|------|
| 懒惰（Lazy） | 😴 | 慵懒，喜欢打盹 |
| 好奇（Curious） | 🔍 | 对一切都感兴趣 |
| 活泼（Playful） | 🎯 | 精力充沛，爱玩 |
| 害羞（Shy） | 🙈 | 内敛，需要慢慢熟悉 |
| 勇敢（Brave） | 🦁 | 大胆，爱探索 |
| 粘人（Clingy） | 🥺 | 非常依赖主人 |

性格影响：AI 日记的写作语气 + AI 聊天的角色扮演风格 + 主页语音气泡文案（6 种性格 × 4 种心情 = 24 种文案变体）

**4 个成长阶段（基于累计专注小时数，与具体习惯无关）**

| 阶段 | 英文 | 累计时间阈值 | 阶段颜色 |
|------|------|------------|---------|
| 幼猫 | Kitten | 0 小时起 | 橙色 |
| 少年 | Adolescent | 满 20 小时 | 绿色 |
| 成年 | Adult | 满 100 小时 | 蓝色 |
| 老年 | Senior | 满 200 小时 | 红色 |

成长阶段基于猫咪绑定的所有专注时间绝对累计，不依赖每日目标完成情况。这意味着：无论每天目标是 15 分钟还是 60 分钟，到达每个阶段需要的绝对专注时间相同，对所有用户公平。

**心情系统（计算值，非存储值）**

心情由「距离上次专注会话的时间」实时计算，不存储在数据库：

| 心情 | 触发条件 | Emoji |
|------|---------|-------|
| 开心（Happy） | 距上次专注 < 24 小时 | 😸 |
| 平和（Neutral） | 1-3 天未专注 | 😺 |
| 孤独（Lonely） | 3-7 天未专注 | 🥺 |
| 想念（Missing） | 超过 7 天未专注 | 😿 |

**主页「特色猫咪」推荐算法**

主页只展示一只「最需要关注的猫咪」，通过加权评分选出：

- 近期活跃度（45%）：最近一次专注的时间越久，分数越高
- 心情紧迫度（30%）：孤独/想念心情的猫咪优先
- 成长进度（20%）：距离下一阶段越近，分数越高
- 今日完成（5%）：今天还没专注的习惯优先

**像素猫外观生成**

每只猫的外观在领养时随机生成，之后永久不变（不可修改、不可重购）。外观由 14 个参数组成：

| 参数类别 | 参数 | 可选数量 |
|---------|------|---------|
| 毛色 | peltType（花纹类型）、peltColor（19 种颜色） | 多种组合 |
| 眼睛 | eyeColor（21 种）、eyeColor2（异色瞳） | 21 种 |
| 特殊纹理 | whitePatches、vitiligo、isTortie（玳瑁）、points（暹罗）| 各若干 |
| 皮肤 | skinColor（鼻子/耳朵颜色，18 种） | 18 种 |
| 体型 | isLonghair（长毛/短毛）、spriteVariant（3 种姿势变体） | — |
| 渲染 | reverse（水平镜像）、tint（色调叠加） | — |

外观不可变的设计理由：创造持久的情感依附，避免用户反复「刷脸」而不形成真实的情感连接。

**猫咪状态**

- 活跃（Active）：正常使用中
- 归档（Archived）：用户手动归档，显示在「猫咪相册」中，可恢复
- 永续模式：习惯目标达成后，猫咪继续积累时间，无上限

### 4.5 AI 功能

**AI 日记（每日猫咪视角日记）**

| 属性 | 值 |
|------|-----|
| 触发时机 | 完成专注后异步触发（fire-and-forget）；打开猫咪详情页时检测今日是否缺失日记 |
| 生成限制 | 每只猫每天最多一条 |
| 输出格式 | 2-4 句，猫咪第一视角，中文以「亲爱的日记，」开头，英文以「Dear diary,」开头 |
| 生成上下文 | 猫咪性格、当前心情、成长阶段、绑定习惯、今日专注分钟、距上次专注时间、累计打卡天数 |
| 失败处理 | 失败记入重试队列（最多 3 次，次日清除）；不影响主流程 |
| 存储位置 | 本地 SQLite（无需网络即可查看历史） |

**AI 聊天（猫咪角色扮演对话）**

| 属性 | 值 |
|------|-----|
| 每日限额 | 每只猫每天最多 5 条用户消息（可通过 Firebase Remote Config 调整） |
| 回复风格 | 保持猫咪角色，1-3 句回复，偶尔使用猫咪拟声词（喵~、呼噜噜），鼓励完成习惯 |
| 流式输出 | Token 逐字显示（typing 效果） |
| 上下文窗口 | 系统 Prompt（~300 tokens）+ 最近 10 轮历史（~1500 tokens）+ 新消息 |
| 失败兜底 | 显示猫咪风格的兜底语，不显示技术错误 |
| 存储位置 | 本地 SQLite |

**AI 可用性断路器**

- 连续 3 次 AI 调用失败 → 进入 5 分钟冷却期
- 冷却期内所有 AI 功能显示「离线」状态
- 5 分钟后自动恢复并重试
- UI 反馈：AppBar 显示 cloud_off 图标

**AI 提供商**

支持两个可切换的 AI 后端：

| 提供商 | 适用场景 |
|--------|---------|
| Firebase Gemini (Google) | 默认提供商 |
| MiniMax | 国内网络环境备选 |

切换方式：设置页 → AI → 选择提供商

**模型测试聊天**

设置页 → AI → 「测试模型」入口，用于验证当前 AI 连接是否正常。内存中运行，不持久化。

### 4.7 配饰商城与经济系统

**金币获取**

| 来源 | 金额 |
|------|------|
| 专注会话完成 | 每分钟 10 金币 |
| 每日签到 | 10-15 金币 / 天 |
| 签到里程碑 | 30-150 金币 / 里程碑 |
| 成就奖励 | 1-2200 金币 / 成就（不等） |

**配饰商城**

- 60+ 件配饰
- 分 3 大类：植物（Plants）、野生（Wild）、项圈（Collars）
- 5 个价格档位：Budget（50 金币）到 Legendary（350 金币）
- UI：3 列 Grid 布局，Tab 切换类别

**配饰使用流程**

```
商城 → 选择配饰 → 确认购买（扣除金币）
    → 进入配饰库存
    → 打开猫咪详情页 → 装备配饰（每只猫同时只能装备一件）
```

**配饰渲染**：装备后，配饰叠加在猫咪像素精灵上方渲染，实时显示效果。

### 4.10 账号与设置

**账号类型**

| 类型 | 说明 |
|------|------|
| 游客模式 | 不需要注册，数据存储在本地 SQLite，功能完整（含 AI） |
| 邮箱账号 | 注册后数据同步到 Firestore |
| Google OAuth | 一键登录，数据同步到 Firestore |

**游客升级流程**

游客使用后选择注册，系统检测云端是否已有数据：
- 云端无数据：静默迁移本地数据到云端
- 云端有数据（曾经注册过）：弹出冲突选择对话框，用户明确选择「保留本地数据」或「保留云端数据」

**账号删除**

- 三步确认流程（防误操作）
- 执行顺序：先删除本地数据 → 队列化远程删除任务 → 后台异步执行云端硬删除
- 删除后自动切换到游客模式

**设置项**

| 设置 | 选项 |
|------|------|
| UI 风格 | Material Design 3 / 像素复古（Retro Pixel） |
| 颜色种子 | 8 种颜色，控制整体色调 |
| 明暗模式 | 浅色 / 深色 / 跟随系统 |
| 语言 | 15 种语言 |
| 通知 | 开启/关闭每日提醒，编辑提醒时间 |
| AI 提供商 | Gemini / MiniMax |
| 账号管理 | 链接邮箱、修改昵称、头像、删除账号 |

## 5. 数据模型

### 5.1 Firestore 集合结构（当前已有）

```
users/{uid}
├── displayName: String
├── email: String
├── coins: int                    // 当前金币余额
├── inventory: List<String>       // 拥有的配饰 ID 列表
├── createdAt: Timestamp
│
├── habits/{habitId}              // 习惯
│   ├── name: String
│   ├── emoji: String
│   ├── goalMinutes: int          // 每日目标分钟数
│   ├── targetHours: int?         // 总目标小时数（null = 永续）
│   ├── reminderTime: String?
│   ├── isActive: bool
│   ├── totalMinutes: int         // 累计专注分钟数
│   ├── totalCheckInDays: int     // 累计打卡天数
│   ├── lastCheckInDate: String?  // 最近打卡日期 'yyyy-MM-dd'
│   ├── targetCompleted: bool     // 总目标是否达成
│   ├── createdAt: Timestamp
│   │
│   └── sessions/{sessionId}      // 专注会话（不可变审计日志）
│       ├── habitId: String
│       ├── catId: String
│       ├── startedAt: Timestamp
│       ├── endedAt: Timestamp
│       ├── durationMinutes: int  // 实际专注分钟数
│       ├── status: String        // 'completed' | 'abandoned' | 'interrupted'

│   ├── totalMinutes: int         // 该猫绑定习惯的累计专注分钟数
│   ├── habitId: String           // 绑定的习惯
│   ├── equippedAccessory: String? // 当前装备的配饰
```

## 6. 技术架构

### 6.1 技术栈

| 层次 | 技术 |
|------|------|
| 前端框架 | Flutter 3.41.x / Dart 3.11.x |
| 状态管理 | Riverpod 3.x（StreamProvider、FutureProvider、StateNotifierProvider） |

| 推送通知 | flutter_local_notifications（本地通知） |
| 像素渲染 | 自研 pixel_cat_maker（Canvas 多层合成） |

### 6.2 依赖流（严格单向）

```
Screens（UI 层）
    ↓ ref.watch / ref.read
Providers（Riverpod 响应式层）
    ↓ 注入
Services（业务逻辑层）
    ↓ 调用
Firebase SDK / SQLite（基础设施层）
```

规则：
- Screen 只能通过 Provider 读取数据，不得直接导入 Service
- Service 不含 BuildContext，不含 Riverpod 引用
- Provider 负责生命周期管理和依赖注入

### 6.3 离线优先架构

```
用户操作
    ↓
写入本地 SQLite（ledger_action + materialized_state）
    ↓ 即时响应 UI
后台 SyncEngine 异步同步到 Firestore
    ↓
Firestore 作为云端备份和多设备同步源
```

效果：App 在无网络环境下功能 100% 可用，同步对用户透明。

### 7.2 像素猫渲染管线

```

    ↓
PixelCatGenerationService（随机生成）
    ↓ 缓存查找（LRU，50 个组合）

    ↓ 水平镜像（如 reverse=true）
PixelCatSprite Widget（展示 + Semantics 可访问性标注）
```

渲染结果 LRU 缓存最多 50 个组合，防止在猫咪列表滚动时重复计算。

## 10. 关键产品决策与设计理由

| 决策 | 理由 |
|------|------|
| **猫咪外观创建后永久不变** | 防止用户反复刷脸而不建立情感连接；类比现实中宠物不会因为「不够好看」就被换掉 |
| **成长阶段基于绝对累计时间（非相对进度）** | 对所有用户公平，无论每日目标多少；鼓励长期投入，而非短期冲刺 |
| **AI 对所有登录用户默认开启（无开关）** | 减少认知负担；AI 日记和聊天是核心体验，不应该被「可选项」稀释 |
| **离线优先（SQLite 为 SSOT）** | 专注过程不应被网络问题打断；同步对用户完全透明 |
| **放弃按钮需要长按** | 防止计时过程中误触；需要明确意图才能中断 |
| **每日签到使用金币奖励，不用 XP** | 两套激励体系分离：XP 代表「专注成长」，金币代表「可用于消费的资产」；签到奖励金币，引导用户去商城消费，形成内循环 |
| **主页只展示一只特色猫咪** | 减少「选择疲劳」；算法自动选出最需要关注的那只，让每次打开 App 都有明确的情感焦点 |
| **双 UI 风格（Material 3 + 像素复古）** | 策略模式实现零代码分支；满足不同审美用户（现代感 vs 怀旧感），对应年轻用户和游戏玩家两类群体 |
