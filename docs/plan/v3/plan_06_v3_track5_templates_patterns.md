# 模块：轨道五 — 丰富度功能（活动 + 清单 + 高光 + 挑战 + 回望）

## 一、模块概述

### 模块目标

交付 LUMI 的 **Phase 3 丰富度功能**，使电子手册完整匹配实体书《点亮内心宇宙》的全部内容。核心交付物：

1. **月度活动 6 主题**：习惯约定、烦恼减负、自我夸奖、支持网络、未来照见、理想 vs 现实
2. **成长回望**：属于我的时刻、总结、小赢颁奖、温暖收官
3. **灵感清单内容**：9 个习惯范例 + ~50 个爱自己活动 + 8 维度成长灵感
4. **清单系统**：书单 / 影单 / 自定义清单 + 年度精选
5. **100 天小赢挑战**：含 25 天里程碑
6. **幸福时刻 + 高光时刻**

### 在项目中的位置

```
轨道一（数据基础）──► 轨道五（丰富度功能）
轨道四（月度/年度）──► 轨道五（清单和高光在年度视图中入口）

轨道五 ─ ─ ─► 旅程 Tab · 探索段（月度活动 6 主题）
轨道五 ─ ─ ─► 旅程 Tab · 年度段（清单 + 高光 + 100 天挑战）
轨道五 ─ ─ ─► 我的 Tab（成长回望入口）
```

### 模块交付物

| 交付物 | 说明 |
|--------|------|
| 月度活动 6 主题 | 旅程 Tab 探索段的 6 个引导式练习 |
| 成长回望 | 半年/年度回顾页面 |
| 清单系统（书单/影单/自定义）| 年度视图中的清单管理 + 年度精选 |
| 100 天小赢挑战 | 年度视图中的长期习惯挑战 |
| 幸福时刻 + 高光时刻 | 年度视图中的美好记忆记录 |

## 三、成长回望

### 3.1 入口

「我的」Tab → `GrowthReview` 卡片入口。Day 90 解锁后可见。

### 3.2 页面结构

```
成长回望
├── 属于我的时刻（自动从幸福/高光时刻中精选 Top 5）
├── 我的总结（自由文本：半年/年度总结）
├── 小赢颁奖典礼（100 天挑战完成统计 + 动画）
└── 温暖的收官（给未来的自己一段话 + 下一个周期邀请）
```

### 3.3 属于我的时刻

- 自动从 `HighlightEntry` 中选取评分最高的 5 条
- 幸福时刻和高光时刻混排，按日期倒序
- 每条显示简要信息 + 点击可查看详情

### 3.4 小赢颁奖典礼

- 展示 100 天挑战的完成统计
- 25 / 50 / 75 / 100 天里程碑是否达成
- 达成的里程碑显示庆祝动画

## 四、灵感清单完整内容

### 4.1 习惯养成范例（9 个）

| 序号 | 习惯 | 看得见 | 想去做 | 易上手 | 有奖励 |
|------|------|--------|--------|--------|--------|
| 1 | 提前 30 分钟睡觉 | 手机闹钟 | 听轻音乐 | 先提前 15 分钟 | 醒来状态更好 |
| 2 | 每天喝 X 升水 | 床头放杯水 | 用喜欢的水杯 | 先喝一杯 | 身体更有能量 |
| 3 | 阅读 X 页书 | 书放在经过的位置 | 选喜欢的书 | 只读五页也行 | 获取知识+放松 |
| 4 | 感恩日记 | 日记本放床头 | 装饰页面 | 太累时写一件事 | 翻阅感受进步 |
| 5 | 姿势提醒 | 便利贴"坐直了！" | 与朋友互相监督 | 看到就调整两秒 | 身体舒服气质提升 |
| 6 | 扔垃圾 | 晚饭后设闹钟 | 播放喜欢的歌 | 只扔一袋 | 干净空间+小奖励 |
| 7 | 整理房间 | 手机闹钟 | 播放音乐 | 只整理一个小区域 | 轻松清爽的环境 |
| 8 | 无电子设备时间 | 设闹钟 | 准备离线活动 | 先两分钟不碰手机 | 睡得更好专注更强 |
| 9 | 洗衣服 | 手机闹钟 | 选喜欢的洗涤剂 | 只洗几件 | 清爽衣服带来肯定 |

### 4.2 爱自己活动（~50 条）

分 5 类：

**自我关怀类**：尝试做新的菜肴、买喜欢的文具、写下自己的优点、对自己说"我爱你"每天至少一次、为自己泡一杯暖暖的茶、清理房间断舍离、参加一次读书会、参观一次艺术展览

**生活享受类**：买一份想吃的零食、拍一张大笑的照片、去一直想去的餐厅、给自己做丰盛的早餐、买漂亮的明信片给自己寄一张、买一束喜欢的花、学习一项新的运动、去海边发呆

**购物犒赏类**：买一套舒适的睡衣、给自己放一天假、写成功日记、去郊外散步呼吸新鲜空气、泡个热水澡加些香薰、整理照片制作相册、参加一次户外野餐、种一盆植物

**美食创意类**：做一顿丰盛的晚餐、买一双舒适的鞋子、学习摄影记录美好瞬间、去唱卡拉 OK 尽情释放、去动物园或植物园游玩、看一部温馨的电影、参加志愿者活动、学习绘画

**探索挑战类**：去参加一次音乐节、尝试拒绝一次、看一场体育比赛、学习剪辑视频、参加一次徒步旅行、尝试射箭、尝试冥想放松身心

### 4.3 年度成长计划灵感（8 维度）

| 维度 | 灵感举例 |
|------|---------|
| 认真学习 | 记笔记更有条理、每天背诵 10 分钟、阅读课外书、学会预习和复习、每天读书 10 分钟 |
| 身心健康 | 每周放松一次、多吃蔬果、保持心情稳定、找适合的运动、少刷手机、按时睡觉 |
| 勇敢自信 | 在众人面前朗读、报名参加新活动、勇敢表达真实想法、接受挑战性任务 |
| 养成习惯 | 每周整理书包、每天读书 10 分钟、早上不赖床、每天喝够水、每晚写一句话日记 |
| 接纳自己 | 没考好也能安慰自己、给自己一个鼓励、不和别人比较太多、记录骄傲的小进步 |
| 创意创作 | 画画或做设计、写一首歌/创作故事、参与创意活动、用影像记录生活 |
| 关系连接 | 多主动和朋友交流、参加集体活动、关心朋友情绪、给亲友写卡片 |
| 善意行动 | 主动帮助同学、关心别人的情绪、参加志愿活动、帮爸爸妈分担家务 |

## 五、清单系统

### 5.1 三种清单

| 清单 | 条目数 | 每条字段 |
|------|--------|---------|
| 书单 | 10 条 | 书名、作者、类型（小说/非虚构/专业/其他）、五星评分、关键词、感受 |
| 影单 | 10 条 | 片名、导演、类型（剧情/喜剧/纪录片/其他）、五星评分、关键词、感受 |
| 自定义清单 | 10 条 | 清单名称（用户自定义）+ 条目名、类型、五星评分、关键词、感受 |

### 5.2 年度精选

每个清单含一个「年度精选」子页面：

- **宝藏推荐**：从 10 条中选出最推荐的 1 条，详细推荐理由
- **灵感一击**：一句话灵感/金句
- **精华墙**：3 条精华摘抄或笔记

### 5.3 数据模型

使用 `UserList` 模型：

```dart
class UserList {
  final String id;
  final String uid;
  final String listType; // 'book', 'movie', 'custom'
  final String? customName; // 自定义清单名称
  final int year;
  final List<ListItem> items; // 10 条
  final YearPick? yearPick; // 年度精选
}
```

## 六、100 天小赢挑战

### 6.1 规则

- 100 天打卡网格（10 x 10）
- 关联一个年度习惯
- 任意 100 天内完成（不要求连续）
- 中断不惩罚：「打了一半也是进步 ✦」

### 6.2 25 天里程碑

| 里程碑 | 天数 | 奖励文案 |
|--------|------|---------|
| 第一阶 | 25 天 | 「四分之一！你正在建立习惯回路 ✦」|
| 第二阶 | 50 天 | 「半程达成！习惯正在内化 ✦」|
| 第三阶 | 75 天 | 「四分之三！你已经很了不起了 ✦」|
| 完成 | 100 天 | 「100 天挑战完成！这是属于你的荣耀 ✦」|

### 6.3 UI 规格

```
┌──────────────────────────────────────┐
│  100 天小赢挑战                       │
│  习惯：阅读 30 分钟                   │
│                                      │
│  ● ● ● ● ● ● ● ● ● ●  ← 第 1 排   │
│  ● ● ● ● ● ○ ○ ○ ○ ○  ← 第 2 排   │
│  ○ ○ ○ ○ ○ ○ ○ ○ ○ ○             │
│  ...                                 │
│  ○ ○ ○ ○ ○ ○ ○ ○ ○ ○  ← 第 10 排  │
│                                      │
│  已完成 15/100 天                     │
│  下一个里程碑：25 天（还差 10 天）     │
```

## 七、幸福时刻 + 高光时刻

### 7.1 数据模型

使用 `HighlightEntry` 模型：

```dart
class HighlightEntry {
  final String id;
  final String uid;
  final String entryType; // 'happy' 或 'highlight'
  final int year;
  final String description; // 发生了什么
  final String? companion; // 和谁一起（幸福时刻）
  final String? myAction; // 我做了什么（高光时刻）
  final String? feeling; // 感受
  final String? date; // 事件日期
  final int? rating; // 1-5 评分
}
```

### 7.2 幸福时刻（10 条）

每条卡片：

```
┌──────────────────────────────────────┐
│  #3                                  │
│  幸福的事：和家人一起看了日落          │
│  和谁一起：爸爸妈妈                   │
│  感受：温暖、平静                     │
│  日期：2026-03-15                    │
│  ◐◐◐◐◑  (4/5)                      │
└──────────────────────────────────────┘
```

- 幸福仪表盘：半圆评分组件（5 级）
- 最多 10 条，可随时编辑

### 7.3 高光时刻（10 条）

每条卡片：

```
┌──────────────────────────────────────┐
│  #2                                  │
│  发生了：完成了第一个开源项目          │
│  我做了：坚持了 3 个月的编码           │
│  感受：骄傲、兴奋                     │
│  日期：2026-02-28                    │
│  🏅                                  │
└──────────────────────────────────────┘
```

- 勋章图标装饰
- 最多 10 条，可随时编辑
