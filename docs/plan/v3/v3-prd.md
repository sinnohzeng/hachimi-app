## 二、用《真需求》框架拆解产品价值

### 2.1 产品价值公式

梁宁：**产品价值 = 功能价值 + 情绪价值 + 资产价值**

| 维度 | LUMI 提供的价值 | 用户的「真需求」|
|------|---------------|--------------|
| **功能价值** | 结构化的时间维度记录模板（日/周/月/年），烦恼处理流程，习惯四法则设计工具 | 需要一个简单到不需要意志力的方式来记录生活 |
| **情绪价值·保障感** | 烦恼被写下来就不再压在心里；烦恼罐把焦虑"外部化" | 对抗半夜翻来覆去的焦虑 |
| **情绪价值·愉悦感** | 发现"原来今天也有好事发生"；看到幸福时刻的规律 | 对抗"浑浑噩噩过了一天"的空虚 |
| **情绪价值·自我认同** | 看到自己在成长的轨迹；星星累积可视化 | 对抗"我不够好"的自我怀疑 |
| **资产价值** | 半年/一年积累的成长档案——属于自己的韧性成长史 | 可回顾、可传承的人生记录 |

### 2.2 第一性原理

> **LUMI 品类的第一性不是"记录"，而是"觉知"。**

记录是手段，觉知是目的。用户真正需要的不是一个写日记的工具，而是一面镜子——让他在每天几行字的微小记录中，渐渐看见自己的情绪模式、幸福来源和成长轨迹。

**我们不是谁**：

| 竞品 | 他们卖的 | 我们的区别 |
|------|---------|----------|
| Notion / Todoist | 功能价值（组织效率）| LUMI 不是效率工具，不做任务管理 |
| Day One | 资产价值（生活档案）| LUMI 有方法论（四法则、三件好事、烦恼外化），不是自由日记 |
| Forest / Habitica | 功能价值（习惯打卡）| LUMI 的习惯是服务于"成长"的子模块，不是独立功能 |
| Calm / Headspace | 情绪价值（冥想放松）| LUMI 通过"写"而非"听"来建立觉知 |
| Finch | 情绪价值（宠物陪伴）| LUMI 的核心是手册方法论，不是养成游戏 |

### 2.3 认知标记设计

梁宁：**情绪价值 = 生理唤起 + 认知标记 + 心理账户**

LUMI 需要在用户心中建立的认知标记：

1. **"韧性不是天生的，是可以练习的"** — 来自张晓萌教授的学术背书（长江商学院副院长、行为心理学教授）
2. **"每天只需要 5 分钟"** — 极低门槛消除抗拒
3. **"在厌倦之前就收笔"** — 反「坚持」叙事，不制造压力
4. **"每写一行，都是在为内心宇宙添一颗星"** — 将微小行动赋予宇宙级意义

**心理账户定位**：LUMI 不在"效率工具"账户里（用户会比性价比），而在"自我关怀"账户里（用户为感觉付费），和冥想 APP、手账本、心理咨询同类。

### 2.4 弱者道之用

梁宁：**"简单"这个词，几乎是所有工具之争的制高点。**

hao123 的启示：不追求功能多、页面漂亮、技术先进，只追求一件事——简单到无法超越。

**LUMI 的设计铁律**：
- 打开 APP → 看到今天的日期 → 写一句话 → 关掉。**10 秒完成**
- 永远不要让用户看到空空的页面（用灵感清单预填充提示）
- 永远不要让用户觉得"我还没写完"（任何页面随时可保存离开）
- 不做功能堆砌的匪兵甲，做有方法论内核的韧性工具
- 每多一个功能，都问：这让用户更简单了，还是更复杂了？

## 三、用户人设（梁宁第七章）

### 3.1 核心人设：「努力但容易自我怀疑的成年人」

| 维度 | 描述 |
|------|------|
| **年龄** | 25–45 岁 |
| **自我期待** | 想变得更好：更自律、更了解自己、内心丰盈 |
| **核心痛点** | 买过很多手账，写了几页就放一边。坚持不下去不是因为不想，而是门槛太高、反馈太慢 |
| **隐性需求** | 不知道「什么真正让自己开心」；烦恼压在心里，半夜翻来覆去 |
| **行为特征** | 手机重度用户；睡前刷手机；时间碎片化 |
| **情感特征** | 渴望温暖但不想被说教，喜欢有质感的东西 |

**人设公式**：
```
用户 = 自我期待（想成为有觉知、有韧性的人）
      + 外部压力（工作忙、时间碎片化）
      + 内在恐惧（害怕一直浑浑噩噩下去、害怕不够好）
```

**产品切入点**：在「自我期待」和「外部压力」之间找到最小可行支点——**每天 5 分钟，足够简单到不会被外部压力挤掉，又足够有效到能触及自我期待。**

### 3.2 非目标用户

- 需要严格 GTD/OKR 时间管理系统的项目经理型用户
- 需要社交打卡、互相监督的外向型用户
- 只想玩养成游戏、不关心自我成长的用户

## 四、原书内容体系与电子化策略

### 4.1 原书《点亮内心宇宙：韧性成长手册》结构

> 详细映射表见 [lumi-book-mapping.md](lumi-book-mapping.md)

实体书以 **6 个月** 为周期，包含 4 大章节：

| 章节 | 页码 | 核心内容 | 时间维度 |
|------|------|--------|---------|
| 年度规划 | 001-026 | 年历、年度寄语（7 问）、成长计划（8 维度）、书单/影单/自定义清单、小赢记录（100 天挑战）、旅行与活动、幸福时刻、高光时刻 | 年 |
| 月度规划 | 027-219 | 月历、月目标+小赢挑战、热爱与坚持（4 习惯追踪+爱自己活动）、心情追踪、本周计划（艾森豪威尔矩阵+烦恼处理器）x4、本周回顾（3 幸福时刻+感恩+学到+随笔+烦恼状态）x4、每日看板（一日一句+时间轴）x4 | 月/周/日 |
| 月度活动 | 220-237 | 6 大主题：习惯约定（四法则）、烦恼减负日、夸夸群、身边的人（支持网络）、未来照见我、理想 vs 现在 | 月（主题轮换）|
| 成长回望 | 238+ | 属于我的时刻、我的总结、小赢颁奖典礼、温暖的收官 | 半年/年 |

附赠**灵感清单**：9 个习惯范例（含四法则设计）、~50 个爱自己活动建议、8 维度成长计划灵感。

### 4.2 心理学方法论内核

这些不只是功能，而是 LUMI 的方法论 IP——来自张晓萌教授的行为心理学研究：

| 方法论 | 原理 | 在 LUMI 中的体现 |
|--------|------|---------------|
| **三件好事** | Seligman 积极心理干预：坚持记录好事的人幸福指数提高 5%，抑郁指数降低 20% | 每周回顾"三个幸福时刻" |
| **烦恼外部化** | 认知行为疗法：写下烦恼 = 从大脑"卸载"到外部存储，降低认知负荷 | 烦恼处理器 + 烦恼罐分类 |
| **习惯四法则** | James Clear 原子习惯：看得见、想去做、易上手、有奖励 | 习惯设计工具 + 小赢挑战 |
| **觉知记录** | 元认知训练：记录行为让人从"自动驾驶"切换到"有意识观察" | 每日一点光 + 心情追踪 |
| **成长型思维** | Dweck：关注过程和进步，而非结果 | "不完美也没关系"设计原则 |

### 4.3 电子化策略：不是搬运，是升维

纸质本的优势：翻开就看到年度计划、随手翻阅、触感温暖。
纸质本的劣势：写错不能改、翻找不方便、无法统计规律。

**电子端的升维设计**：

| 纸质本局限 | 电子端升维 |
|-----------|----------|
| 页面固定，6 个月用完 | 无限循环，每月自动生成新页 |
| 翻阅找内容不方便 | 搜索 + 按时间维度快速切换 |
| 无法看到情绪规律 | 心情热力图 + 标签统计 + 模式发现 |
| 写错无法修改 | 随时编辑、保存草稿 |
| 灵感清单是附赠小册子 | 内嵌提示词，空页面自动显示灵感 |
| 100 天挑战需要手动划格子 | 自动打卡 + 里程碑动画 + 奖励机制 |

**但也要保留纸质本的温度**：
- 手写体风格的提示语（"我想更专注在……上"）
- 章节分隔页的仪式感（云朵气泡 + 猫咪插图）
- 低密度信息展示（留白 > 堆砌）
- "在厌倦之前就收笔"的零压力体验

## 五、导航架构：3 Tab

### 5.1 战略决策：从 4 Tab 减到 3 Tab

遵循"弱者道之用"——更少的选择 = 更低的认知负荷。

**旧（v2.x / v3 草案）**：觉知 | 习惯 | 猫咪 | 我的（4 Tab）

**新（LUMI）**：

```
┌─────────────────┬─────────────────┬─────────────────┐
│     ✦ 今天      │    🗺 旅程      │    👤 我的      │
│    Record       │    Journey      │    Profile      │
└─────────────────┴─────────────────┴─────────────────┘
```

### 5.2 各 Tab 职责

| Tab | 核心功能 | 对应原书 |

| **今天** | 每日一点光（心情+一句话）+ 今日习惯快照 + 灵感提示 | 每日看板"一日一句" |

| **我的** | LUMI 统计 + 猫咪伴侣入口（二级）+ 设置 | — |

### 5.3 猫咪与习惯的新位置

| 功能 | 旧位置 | 新位置 |
|------|-------|-------|
| 猫咪领养 | 强制首关 | 删除，变为"我的"里的可选入口 |

## 六、渐进解锁：不给用户压力的功能展开

> "再记录 X 天，就可以开始年度规划了。不急，慢慢来 ✦"

## 八、场景设计：四维时间结构

从原书的三幕结构扩展为完整的四维时间体系。

### 8.1 每日：睡前一点光（~30 秒 - 2 分钟）

**触发**：用户设定的睡前提醒

| 模块 | 说明 | 最简路径 |
|------|------|---------|
| 心情选择 | 5 个 emoji（很开心→很低落）| 1 次点击 |
| 今日一点光 | "今天有什么让你微笑的事？" | 一句话（≤ 500 字，可为空）|
| 标签（可选）| #家人 #学习 #户外 等预设标签 | AI 模式发现的数据基础 |
| 习惯打卡（可选）| 今日小赢挑战完成状态 | 快速勾选 |

**最少路径**：选 emoji + 写一句话 = **30 秒完成**

### 8.2 每周：周日复盘（~5 分钟）

**触发**：周日 20:00 温柔推送

**周计划（周一早）**：

| 模块 | 说明 |
|------|------|
| 一句话给自己 | 本周的一个小心愿 |
| 艾森豪威尔四象限 | 必须做 / 要做 / 该做 / 想做，各含时长估算 |
| 烦恼记录 | 新增本周烦恼 + 解法 |

**周回顾（周日晚）**：

| 模块 | 说明 |
|------|------|
| 三个幸福时刻 | 3 个编号卡片，可只填 1-2 个 |
| 感恩 | 这周我想感恩…… |
| 学到了什么 | 这周我学到了…… |
| 随笔 | 自由记录区 |
| 烦恼状态更新 | 自然消失 / 自己搞定 / 还在 → 烦恼罐 |

### 8.3 每月：月度仪式（~5 分钟）

**触发**：每月 1 日温柔提示

| 模块 | 说明 |
|------|------|
| 本月目标 | 5 个目标条目 |
| 小赢挑战 | 选一个习惯 + 31 天打卡 + 四法则设计 + 奖励 |
| 爱自己活动 | 本月要做的开心事清单（灵感清单预填） |
| 小习惯追踪 | 最多 4 个习惯 x 31 天网格 |
| 本月回顾（月末）| 本月记忆 + 本月成就 |

### 8.4 年度：年度规划（随时维护）

**触发**：记录 14 天后温柔邀请

| 模块 | 说明 |
|------|------|
| 年度寄语 | 7 个提问：希望成为/达成目标/突破性完成/不做/年度关键词/给未来的我/座右铭 |
| 成长计划 | 8 维度：认真学习/身心健康/养成习惯/勇敢自信/接纳自己/创意创作/关系连接/善意行动 |
| 年历 | 12 月 x 31 天重要日期标记 |
| 清单 | 书单/影单/自定义清单，各 10 条 + 年度精选（宝藏推荐 + 灵感一击 + 精华墙）|
| 小赢 100 天 | 100 天挑战，25/50/75/100 天里程碑 + 奖励 |
| 幸福时刻 | 10 条：幸福的事 + 和谁一起 + 感受 + 日期 + 幸福仪表盘 |
| 高光时刻 | 10 条：发生了 + 我做了 + 感受 + 日期 + 勋章 |
| 旅行与活动 | 12 预设 + 4 自定义：时间/地点/感受/五星评分 |

## 九、KANO 属性分析

### 必备属性（不可缺少）

- 离线可用（SQLite SSOT）
- 数据安全（Firestore 云同步）
- 随时保存、随时退出（不丢数据）
- 中文界面优先

### 期望属性（MVP 核心交付）

- 每日一点光（心情 + 一句话 + 标签）
- 周计划与周回顾
- 烦恼处理器
- 月度目标 + 小赢挑战
- 渐进解锁

### 魅力属性（有就惊喜）

- 星星累积可视化（每写一行 +1 星）
- AI 模式发现（30+ 天后："你幸福时刻 65% 和 #户外 有关"）
- 心情热力图（月历颜色可视化）
- 猫咪温暖回应（情绪感应）
- 年度成长报告

### 反向属性（绝对不做）

- ❌ 催促式通知（"你今天还没打卡！"）
- ❌ 断连惩罚（连续天数中断显示红色警告）
- ❌ 社交分享/排行榜（觉知是私密的）
- ❌ 强制每日打开
- ❌ 空页面无引导（必须有灵感提示）

## 十、「不完美也没关系」设计原则

来自播客：「打了一半的卡，有半个月不熬夜也是进步。」

| 场景 | 常规做法 | LUMI 做法 |
|------|---------|----------|
| 连续天数中断 | 红色警告 / 归零 | "中断了也没关系，今天又是新的开始 ✦" |
| 周回顾不完整 | 不允许保存 | 幸福时刻可只填 1 个，随时保存 |
| 月度打卡未满 | 显示"未完成" | "这个月打了 X/30 天，每一天都算数 ✦" |
| 烦恼未解决 | 负面标记 | "还在的小烦恼——慢慢来" |
| 年度计划没写完 | 空白焦虑 | 灵感清单自动提示，逐步填充 |

## 十一、通知策略

| 类型 | 触发 | 时间 | 文案 |
|------|------|------|------|
| 睡前一点光 | 用户设定时间 | 每晚 | "今天的一点光，在等你 ✦" |
| 周日复盘 | 每周日 | 20:00 | "这周辛苦了，来回顾一下吧 ✦" |
| 月初仪式 | 每月 1 日 | 10:00 | "新的一月，设个小目标？✦" |

**绝对不做的通知**：
- ❌ "你今天还没打卡！"
- ❌ "你的连续天数即将中断！"
- ❌ 每日弹窗催促

## 十三、猫咪系统退场与过渡

### 13.1 战略决策

猫咪从"产品核心"降级为"可选伴侣"：
- 不再是入口门槛
- 不再占据一级 Tab
- 作为温暖的情感附加值存在

### 13.2 过渡计划

| 阶段 | 猫咪状态 | 说明 |
|------|---------|------|
| Phase 1 | 移除强制领养，退到"我的"二级入口 | 用户可选择是否领养 |
| Phase 2 | 猫咪对每日一光做出温暖回应 | 从"主角"变为"配角" |
| 未来（商业化前）| 替换不可商用的猫咪图标组件 | 用自研 LUMI 吉祥物替代 |

### 13.3 习惯系统重新定位

习惯不再是独立功能，而是韧性手册"本月热爱与坚持"的子模块：
- 小赢挑战 = 月度习惯重点
- 习惯追踪 = 月度 4 个小习惯网格
- 专注计时 = 习惯详情页的深层入口

## 十六、关键设计原则速查

| 原则 | 来源 | 在产品中的体现 |
|------|------|-------------|
| 每天只要 5 分钟 | 张晓萌 | 核心交互 ≤ 30 秒可完成 |
| 门槛低到不需要「坚持」 | 张晓萌 | 最少操作 = 选心情 + 写一句话 |
| 在厌倦之前就收笔 | 原书 | 任何页面随时可保存离开 |
| 弱者道之用 | 梁宁 | 3 Tab 不是 5 Tab，简单不是简陋 |
| 不做匪兵甲 | 梁宁 | 方法论内核（四法则/三件好事/烦恼外化）是护城河 |
| 关怀而非操纵 | 梁宁 | 不催促、不惩罚、不制造焦虑 |
| 情绪价值 > 功能价值 | 梁宁 | 卖的是"觉知和成长的感觉"，不是"记录效率" |
| 不完美也没关系 | 张晓萌 | 中断不惩罚，少打卡也认可进步 |
| 记录→觉察→成长 | 原书 | 三阶段成长路径贯穿全 APP |
| 每写一行添一颗星 | 原书 | 星星累积是核心动画和品牌符号 |
